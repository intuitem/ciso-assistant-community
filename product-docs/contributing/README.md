# Contributing

CISO Assistant is open source and improves through community contributions. There are four main paths in, depending on what you want to add:

- [**Frameworks and libraries**](framework.md) — submit a new framework, mapping, threat catalogue, risk matrix, or reference-control set.
- [**Code (features and fixes)**](code.md) — bug fixes, performance improvements, new capabilities. Dev setup, conventions, PR flow.
- [**Translations**](translation.md) — translate the interface into a new locale or improve an existing one.
- [**Documentation**](documentation.md) — fix or extend the pages you're reading right now.

All contributions go through the same [community repository](https://github.com/intuitem/ciso-assistant-community) and pull-request flow. First-time contributors will be asked to sign the [Contributor License Agreement](https://github.com/intuitem/ciso-assistant-community/blob/main/Contributor%20License%20Agreement.md) when their first PR is opened.

If you've found a security vulnerability, **please don't open a public issue** — follow the [security policy](https://github.com/intuitem/ciso-assistant-community/blob/main/SECURITY.md) instead.
